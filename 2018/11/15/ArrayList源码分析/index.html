ArrayList源码分析

发表于 2018-11-15

概述

ArrayList 是一种变长的集合类，基于Object数组实现。
ArrayList 允许空值和重复元素，当往 ArrayList 中元素数量大于其底层数组容量时，其会通过扩容机制重新生成一个更大的数组，然后把旧数组的元素复制到新数组中。
另外，由于 ArrayList 底层基于数组实现，所以其可以保证在 O(1) 复杂度下完成随机查找操作。
其他方面，ArrayList 是非线程安全类，并发环境下，多个线程同时操作 ArrayList，会引发不可预知的错误。

源码分析

先看类的定义，ArrayList类继承AbstractList接口，实现了List、RandomAccess、Cloneable接口。
也就是说，ArrayList是List的实现类，支持高效随机访问，能被克隆和序列化。

构造方法

我们关注两个比较常见的构造器：空参构造器和需要传入默认容量的构造器

空参构造器

可以发现，我们用空参构造器创建ArrayList对象的时候，只是让内部的Object数组引用elementData指向了之前定义的空数组。那么为什么说ArrayList的默认容量是10呢？
别急，当我们去看add()方法的流程，会有意外惊喜！

简单描述一下：

  1. 用空参构造器创建一个ArrayList， elementData =DEFAULTCAPACITY_EMPTY_ELEMENTDATA = {}； size = 0;
  2. 当调用add(E e)方法往ArrayList中添加第一个元素的时候，就会调用 ensureCapacityInternal(size + 1)，此时传入的ensureCapacityInternal(1)。
  3. calculateCapacity()方法去判断这个elementData，发现它指向DEFAULTCAPACITY_EMPTY_ELEMENTDATA，于是 return Math.max(DEFAULT_CAPACITY, minCapacity)， 此时DEFAULT_CAPACITY=10，minCapacity=1。所以calculateCapacity()方法返回的就是10。
  4. 这时执行到了ensureExplicitCapacity(10)，if语句判断完毕，调用grow()方法，此时传入的是10。grow()就是ArrayList的扩容方法，在这里elementData = Arrays.copyOf(elementData, newCapacity); 在这个时候才为Object数组分为了10个容量的内存空间。

有参构造器

可以发现，如果我们用有参构造器，在创建ArrayList的时候就指定了容量，那么内部是直接new出对应容量的Object数组，不会有扩容的操作。

这两个构造函数做的事情很简单，就是初始化底层数组elementData。
区别在于，无参构造函数就是让elementData指向一个空数组，在第一次插入元素的时候，要进行扩容操作，创建一个容量为10的Object数组，再让elementData指向这个数组。
有参构造函数，在创建ArrayList对象的时候，elementData就指向了一个new好对应容量的Object数组。在第一次添加元素的时候，底层不会有扩容的操作。

我们知道了这两个构造函数的区别，对我们写代码有什么指导意义吗？
如果我们知道要用ArrayList保存多少个元素，那么我们在创建ArrayList的时候，在有参构造函数上就要指定好数组的长度。
因为如果用空参构造函数创建ArrayList，数组长度只有10，添加大量元素的过程中，可能就需要进行多次扩容操作，扩容操作涉及到新数组的创建和新旧数组的复制，是比较消耗性能的。

插入

因为ArrayList底层基于数组实现，所以插入元素的时候，可以插入到元素序列末尾，也可以插入到元素序列中间。

如果只是在元素序列末尾插入元素，只要判断数组空间是否足够，然后把新元素放到元素序列末尾就可以了。

如果是在某个合理的元素序列中间的指定位置插入元素，那么要先判断数组空间是否足够，把该位置和后面位置的元素全部往后移动，再把新元素放到指定位置上。这很明显是一个时间复杂度为O(n)的操作，如果数组元素频繁的移动，会影响程序的运行效率。

删除

删除方法不同于添加方法，没有空参的删除方法，只能指定删除某个位置的元素或者某个元素。

可以发现，我们删除的时候一般都是在删除元素序列中间的某个元素，那么被删除元素后面的所有元素又都要往前移动一位。这依然是个时间复杂度为O(n)的操作。

假如我们添加了很多元素，然后又删除很多元素。那么底层Object数组的容量会很大，但大部分空间并没有被使用。

查找

可以基于下标索引进行查找：get(int index)方法

索引范围检查完毕后直接返回对应位置的元素就好了。

但有的时候我们不知道元素的下标索引是什么，就只能通过元素本身来找了。
ArrayList提供了indexOf和lastIndexOf方法，它们都是遍历查找，找到后返回元素在数组中的索引。

从这个indexOf方法里面，我们也可以发现，ArrayList是可以保存null的。

至于其他元素，则是遍历数组中的每个元素，用equals()方法进行判断。

那么你可能会有疑问，既然indexOf就可以查找元素了，那么为什么ArrayList又提供了lastIndexOf方法呢？

因为ArrayList支持保存重复元素啊！

总结

ArrayList是我们经常使用的一个集合类，内部维护一个Object数组，数组本身是定长的，当容量不够时，我们会创建一个容量为原来1.5倍的新数组，然后把旧数组的元素复制到新数组中。
ArrayList基于数组实现，所以可以通过下标索引直接找到对应的元素，查找的时间复杂度是O(1)，但插入或删除元素在一般情况下需要移动大量元素，时间复杂度是O(n)。
